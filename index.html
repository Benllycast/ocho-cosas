funciones ternarias

Def.

Es una función que evalúa una condición y dependiendo de su validez ejecuta una de dos expresiones.

Uso

<condición> ? <si verdadero> : <si falso>;

Ejemplo

var horas = (new Date()).getHours(),
		periodo = horas > 12 ? 'pm' : 'am';

Uso innecesario

var x = (1 < 2) ? true : false;
var x = (1 < 2);

Uso incorrecto

var x = ( Math.random() > 0.5 && window.ontouchend ) ? document.addEventListener('click', function(evt){ var z = evt.target.offsetHeight }) : document.addEventListener('touchend', function(evt){ var z = evt.target.offsetHeight });

coerción

Def.

Es forzar el cambio de un tipo a otro, por ejemplo: cambiar de un número a una cadena de texto.

Temor

Algunos temen el uso de la coerción y la tildan como mala práctica, e incluso como peligrosa.

Pero

La coerción se encuentra naturalmente en JavaScript.

De hecho es natural en muchos lenguajes.

Muchas veces pasa implícitamente

Cuando verificas si una variable existe

if( variable ){}
// JavaScript lo coerce a booleano
// es equivalente a:
if( Boolean(variable) ){}
// o...
if( !!variable ){}

Cuando accedes una propiedad sin string

myObj[ 3 ]; 
// JavaScript lo coerce a string
myObj[ '3' ]
// Yep, arreglos tambien aplican
[ 1, 2, 3 ][ '2' ]  //  3

Cuando concatenas... (no tan implícito)

var edad = 20;
console.log( "Tu edad es: " + edad ); 
// JavaScript coerce edad<number> a string
console.log( "Tu edad es: " + "20" )

Cuando usas isNaN

...isNaN

isNaN( '3' )
// +'3' -> 3
// entonces
> false

isNaN( '3px' )
// +'3px' -> NaN
> true

...isNaN (2)

isNaN( '' )
// +'' -> 0
> false

isNaN( {} )
// +{} -> NaN
> true

...isNaN!! (3)

isNaN( { valueOf: function(){ return 2 }} )
//+{ valueOf: function(){ return 2 }} -> 2
> false

isNaN( { valueOf: function(){ return '' }} )
// +{ valueOf: function(){ return 0 }} -> +'' -> 0
> false

Cuando usas Array.prototype...

Diferentes funcionalidades de los arreglos como: .filter, .every, .some, esperan que la funcion pasada retorne un valor coercionable a booleano.

O explícito...

Cuando usas un contructor de tipo core como función, es decir, sin new

Boolean( 19 ) // true
String( 19 ) // '19'
Number( '19' ) // 19
Number( true ) // 1

O incluso con new estás coercionando.

Coerción en fechas

var date = new Date()
date.getTime() // 1367795189000
date.valueOf() // 1367795189000
+ date // 1367795189000
date.toString() // "Sun May 05 2013 18:06:29 GMT-0500 (COT)"
date + '' // "Sun May 05 2013 18:06:29 GMT-0500 (COT)"

Coerción en strings

var text = '123'
parseInt( text ) // 123
Number( text ) // 123
+ text // 123

Coerción en strings (2)

var text = '34.5%'
Number( text ) // NaN
+ text // NaN
parseInt( text ) // 34
parseFloat( text ) // 34.5

operadores binarios

Operadores binarios

Los operadores binarios son comunmente usados para interacturar con valores booleanos o binarios, no obstante, son aplicables a otros tipos, pues todo se representa en bits.

Evil!

El uso de operadores binarios fuera de booleanos o valores binarios se presta para:
a) Errores de cálculo
b) Difícil lectura del código

Usos comúnes

if( a && b) // en condicionales
var c = c || a // asignaciones
true == 4 // validaciones
var c = a | b // operaciones binarias.. 1 | 2 = < 01 | 10 = 11 > = 3
for(i=0; i<arr.lemgth; i++, j++) // coma (también operador binario) en ciclos

Otros usos no tan comunes

if( a && b) // en condicionales
var c = c || a // asignaciones
true == 4 // validaciones
var c = a | b // operaciones binarias.. 1 | 2 = < 01 | 10 = 11 > = 3
